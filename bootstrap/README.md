# Bootstrap

## Structure

- `helmfile.d/00-crds.yaml`: Extracts CRDs from charts before app install.
- `helmfile.d/01-apps.yaml`: Installs core cluster apps via Helmfile.
- `helmfile.d/templates/values.yaml.gotmpl`: Reads HelmRelease values to
  keep bootstrap Helmfile in sync with Flux-managed HelmReleases and expands
  `${SECRET}` placeholders using cluster secrets.
- `resources.yaml.j2`: Bootstrap-only resources seeded from 1Password via
  `op inject` (applied by `scripts/bootstrap/apply_bootstrap_resources.sh`);
  notably the ESO `onepassword` Connect token.

## Prerequisites

- `kubectl`, `helmfile`, `kustomize`, `yq`, `sops`, `op` (1Password CLI), `minijinja-cli`
- A working k3s cluster with kubeconfig at `./kubeconfig` (generated by Ansible).
  (_Note: `kube-vip` and initial `cilium` are default-installed by `k3s-install` playbook_)
- An authenticated 1Password CLI session (`op signin`, or export `OP_SERVICE_ACCOUNT_TOKEN`).

## Usage

The whole sequence below is wrapped by `just flux bootstrap`; run it end-to-end
with that recipe, or run the individual steps manually as shown.

1. Wait for nodes to become ready.

2. Apply namespace manifests:

   ```sh
   bash ./scripts/bootstrap/apply_namespaces.sh
   ```

3. Apply the SOPS bootstrap secret.
   This seeds the `sops-age` decryption key into `flux-system` — the one credential Flux needs for its own in-cluster SOPS decryption (git sync is anonymous over HTTPS against the public repo, so no deploy key is required):

   ```sh
   bash ./scripts/bootstrap/apply_sops_secrets.sh
   ```

4. Apply the 1Password bootstrap resources.
   These seed the ESO `onepassword` Connect token (so External Secrets can authenticate) plus other bootstrap-time secrets, rendered from `bootstrap/resources.yaml.j2` via `op inject`:

   ```sh
   bash ./scripts/bootstrap/apply_bootstrap_resources.sh
   # equivalent to:
   # minijinja-cli "bootstrap/resources.yaml.j2" | op inject | kubectl apply --server-side -f -
   ```

   > Steps 3 and 4 are complementary, **not** alternatives: SOPS seeds Flux's own
   > credentials; `op inject` seeds the credentials ESO/1Password needs.

5. Install CRDs via Helmfile.

   ```sh
   bash ./scripts/bootstrap/apply_crds.sh
   ```

6. Install core apps via Helmfile.

   ```sh
   bash ./scripts/bootstrap/sync_hrs.sh
   ```

7. Confirm initial deployment and cluster rollout.
